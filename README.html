MongoDB Agent AI Documentation

1. Overview

The MongoDB Agent AI bridges your MongoDB databases with AI-powered development tools like VS Code Copilot, Cursor, and Claude Desktop. Once configured, it enables these tools to interact with MongoDB data in real-time, bringing database querying directly into your coding workflow.

This solution helps you:

  • Query and analyze MongoDB data using natural language without writing aggregation pipelines
  • Create intelligent queries based on natural language commands with automatic optimization
  • Bulk process tasks like generating reports from meeting notes or specifications
üí° Key Insight: MongoDB Agent AI converts plain English questions into optimized MongoDB aggregation pipelines, returning results in seconds. No more context switching between tools or memorizing query syntax.

2. Target Audience

This tool is designed for:

  • Developers who need quick MongoDB data access without leaving their IDE
  • Data Analysts who want to explore data using natural language
  • DevOps Engineers managing MongoDB deployments and need operational insights
  • Product Managers who need ad-hoc database queries for analytics
  • Technical Writers documenting database schemas and usage patterns

3. Installation

3.1 Prerequisites

  • Python 3.10+ Required
  • MongoDB 4.0+ (Atlas, self-hosted, or on-premise)
  • LLM API Key (OpenAI, Anthropic, or Azure OpenAI)
  • 5 minutes for Direct Connection setup

3.2 Quick Setup

Step 1: Download the distribution package

cd mongodb_agent_distribution_general

Step 2: Run the setup script

chmod +x setup.sh
./setup.sh

The script will:

  • ‚úÖ Check Python version
  • ‚úÖ Create virtual environment
  • ‚úÖ Install dependencies
  • ‚úÖ Generate configuration file

Step 3: Configure environment variables

# Edit .env file
nano .env

# Add MongoDB connection
MONGODB_CONNECTION_TYPE=direct
MONGODB_URI=mongodb+srv://user:pass@host/
MONGODB_DATABASE=your_database

# Add LLM provider
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-your-key-here

Step 4: Start the server

python3 -m mongodb_agent.api

# Server runs on http://localhost:8000

3.4 Available Features (15+ Operations)

Query Operations:

  • Natural language to MongoDB aggregation pipeline conversion
  • Count, filter, sort, and limit operations
  • Group by and aggregation functions (sum, avg, min, max)
  • Date range queries with intelligent parsing
  • Nested document field access

Connection Types:

  • Direct Connection - PyMongo driver with connection pooling (~10ms latency)
  • MCP Protocol - Model Context Protocol for Claude Desktop integration (~150ms latency)

LLM Providers:

  • OpenAI (GPT-4, GPT-3.5)
  • Anthropic (Claude 3 Opus, Sonnet)
  • Azure OpenAI

3.5 Troubleshooting

Connection failed?

# Test MongoDB connection
mongosh "your-connection-string" --eval "db.adminCommand('ping')"

Authentication error?

  • Verify credentials in .env file
  • Check authSource parameter in connection string
  • Ensure user has read permissions

Module not found?

# Ensure virtual environment is activated
source venv/bin/activate

# Reinstall package
pip3 install --force-reinstall mongodb_agent_ai-1.0.0-py3-none-any.whl

3.6 FAQ For This Agent

Q: Can I use this with MongoDB Atlas?

A: Yes! Just use your Atlas connection string in MONGODB_URI.

Q: Does this work with on-premise MongoDB?

A: Absolutely. Supports self-hosted MongoDB 4.0+.

Q: What's the difference between Direct and MCP connections?

A: Direct is faster (10ms) and simpler. MCP is for Claude Desktop integration with centralized auth.

Q: Can I use my own LLM model?

A: Yes, supports OpenAI, Anthropic, and Azure OpenAI. More providers coming soon.

Q: Is my data secure?

A: Yes. Uses TLS encryption for MongoDB. LLM queries are sent securely. Never stores credentials.
