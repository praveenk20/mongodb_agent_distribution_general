MongoDB Agent AI - Troubleshooting

üîß Troubleshooting

Common issues and solutions for MongoDB Agent AI.

üîå Connection Errors

‚ùå Error: "Cannot connect to MongoDB"

Symptoms:

ERROR: Cannot connect to MongoDB
ConnectionError: [Errno 61] Connection refused
‚úÖ Solution:
  1. Verify MongoDB is running: mongosh --host your-host --eval "db.adminCommand('ping')"
  2. Check connection string in .env: 
    MONGODB_URI=mongodb+srv://user:pass@host/?authSource=admin
  3. Test network connectivity: ping your-mongodb-host
  4. Verify firewall rules allow connection on port 27017
  5. Check MongoDB authentication: ensure username/password are correct

‚ùå Error: "Authentication failed"

Symptoms:

ERROR: Authentication failed
pymongo.errors.OperationFailure: Authentication failed
‚úÖ Solution:
  1. Verify credentials in .env file
  2. Check authSource parameter in connection string: 
    MONGODB_URI=mongodb+srv://user:pass@host/?authSource=admin
  3. Test credentials with mongosh: 
    mongosh "mongodb://user:pass@host/admin"
  4. Verify user has proper roles: db.getUser("username")
  5. URL-encode special characters in password

‚ùå Error: "Cannot connect to MCP endpoint"

Symptoms:

ERROR: Failed to connect to MCP server
ConnectionError: HTTPConnectionPool(host='localhost', port=3000)
‚úÖ Solution:
  1. Verify MCP server is running: 
    curl http://localhost:3000/health
  2. Check MONGODB_MCP_ENDPOINT in .env
  3. Test MCP endpoint manually: 
    curl -X POST http://localhost:3000/mongodb/query \
  -H "Content-Type: application/json" \
  -d '{"query": "test"}'
  4. Verify OAuth token is valid (check logs for token acquisition)
  5. Fallback: Switch to Direct Connection: 
    MONGODB_CONNECTION_TYPE=direct

‚ùå Error: "Database not found"

Symptoms:

ERROR: Database 'your_db' does not exist
‚úÖ Solution:
  1. List available databases: 
    mongosh "your-connection-string" --eval "show dbs"
  2. Update MONGODB_DATABASE in .env with correct name
  3. Verify user has access to database: 
    db.getCollectionNames()

ü§ñ LLM Provider Errors

‚ùå Error: "Invalid API key"

Symptoms:

ERROR: OpenAI API error: Invalid API key
AuthenticationError: Incorrect API key provided
‚úÖ Solution:
  1. Verify API key in .env: 
    OPENAI_API_KEY=sk-...your-actual-key...
  2. Ensure no extra spaces or quotes around key
  3. Test API key: 
    curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
  4. Check API key permissions and usage limits
  5. Generate new API key if needed

‚ùå Error: "Rate limit exceeded"

Symptoms:

ERROR: Rate limit exceeded
RateLimitError: You exceeded your current quota
‚úÖ Solution:
  1. Check your API usage dashboard
  2. Wait for rate limit reset (usually 60 seconds)
  3. Implement request throttling in your application
  4. Upgrade API plan if consistently hitting limits
  5. Consider switching LLM providers temporarily

‚öôÔ∏è Configuration Errors

‚ùå Error: ".env file not found"

Symptoms:

WARNING: .env file not found, using defaults
ERROR: Required environment variables missing
‚úÖ Solution:
  1. Verify .env file exists in current directory: 
    ls -la .env
  2. Create .env if missing: 
    cp .env.template .env
  3. Ensure you're running server from correct directory: 
    cd /path/to/mongodb_agent_distribution_general
python3 -m mongodb_agent.api

‚ùå Error: "Module not found"

Symptoms:

ModuleNotFoundError: No module named 'mongodb_agent'
‚úÖ Solution:
  1. Verify virtual environment is activated: 
    source venv/bin/activate
  2. Check package installation: 
    pip3 list | grep mongodb-agent-ai
  3. Reinstall if needed: 
    pip3 install --force-reinstall mongodb_agent_ai-1.0.0-py3-none-any.whl

üìä Query Errors

‚ùå Error: "Collection not found"

Symptoms:

ERROR: Collection 'your_collection' does not exist
‚úÖ Solution:
  1. List available collections: 
    mongosh "your-connection-string" \
  --eval "db.getCollectionNames()"
  2. Verify collection_name parameter in API request
  3. Check case sensitivity (MongoDB collections are case-sensitive)
  4. Ensure user has read permissions on collection

‚ùå Error: "Invalid query generated"

Symptoms:

ERROR: Generated query is invalid
pymongo.errors.OperationFailure: $match stage expected
‚úÖ Solution:
  1. Rephrase your question more clearly
  2. Provide more context in the question
  3. Check semantic model for collection (if using YAML models)
  4. Enable DEBUG logging to see generated query: 
    LOG_LEVEL=DEBUG
  5. Review query in logs and manually test in MongoDB

üöÄ Performance Issues

‚ö†Ô∏è Slow query execution

Symptoms:

Queries take more than 10 seconds to complete

‚úÖ Solution:
  1. Check MongoDB indexes: 
    db.collection.getIndexes()
  2. Add indexes for frequently queried fields
  3. Limit result set size in questions: "Show me top 10..."
  4. Use Direct Connection instead of MCP for lower latency
  5. Enable query profiling: 
    db.setProfilingLevel(2)
  6. Review slow query logs
